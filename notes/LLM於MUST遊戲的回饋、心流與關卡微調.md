# LLM於MUST遊戲的回饋與關卡微調的心流把控

## 問題意識

在 MUST 遊戲中，LLM 的角色不應只停留在「會玩」。

如果這類遊戲要真正形成一個可持續優化的系統，LLM 除了能玩、能寫攻略、能做直播解說之外，還應該能對遊玩體驗本身提出回饋，並參與關卡流程的微調。

這裡最重要的一個設計標準，就是：

**心流（Flow）**

也就是玩家在遊玩過程中，是否處在一種：

- 挑戰足夠
- 不會過度混亂
- 理解逐步建立
- 壓力與成就感平衡
- 挫折能轉化為學習

的狀態。

對 MUST 遊戲來說，這件事尤其重要。因為這類遊戲本來就有：

- 多單位同步決策
- 高聯合動作複雜度
- 不完全資訊
- 高語意規則密度
- 協同、欺敵、預判等高階博弈

如果關卡節奏沒有設計好，玩家很容易不是感到「有挑戰」，而是感到「太亂、太難懂、太早挫敗」。

---

## 攻略與心得：兩種不同的輸出

若要讓 LLM 幫助關卡調整，至少需要兩種不同性質的輸出：

### 1. 攻略

攻略回答的是：

> 這個遊戲或這一關，應該怎麼玩會比較好？

它偏向策略知識，會包含：

- 對機制的理解
- 對敵方行為的判斷
- 對局勢的處理原則
- 對後續重玩可直接複用的建議

攻略優化的是：

**勝率、可複用性、策略品質**

### 2. 心得

心得回答的是：

> 這一關玩起來的感受是什麼？

它偏向體驗回饋，會包含：

- 哪裡理解負擔過高
- 哪裡壓力恰到好處
- 哪裡節奏拖沓或突兀
- 哪裡失敗後有學習感
- 哪裡失敗後只覺得莫名其妙

心得優化的是：

**體驗、節奏、可理解性、心流**

這兩者不能互相取代。  
只看攻略，會知道怎麼贏，但不一定知道玩家玩起來痛不痛苦。  
只看心得，會知道感受，但不一定知道如何把挑戰保留又調整得更合理。

---

## 為什麼 MUST 遊戲特別需要心得回饋

在一些簡單 RL 遊戲裡，難度設計相對透明。

例如：

- Flappy Bird：飛不過去就是太難
- Maze：找不到出口就是探索效率不足
- CartPole：桿子容易倒就是控制不穩

但在 MUST 遊戲中，玩家受挫的原因可能非常複雜：

- 不是因為太難，而是因為規則資訊太密
- 不是因為敵人太強，而是因為新機制一次出現太多
- 不是因為策略不足，而是因為回饋不夠清楚
- 不是因為理解不了，而是因為節奏缺乏緩衝

這些問題，單看勝率、失敗率、回合數，通常抓不太出來。

必須要有一種能把體驗轉譯成語言的機制，而 LLM 正好適合做這件事。

---

## LLM 可以提供哪些體驗回饋

一個負責遊玩 MUST 關卡的 LLM，在每次遊玩後，除了輸出攻略，還可以輸出心得。

這類心得可以包含：

- 我在哪一關第一次覺得資訊量明顯過高
- 我在哪一關開始感到規則學習和戰術應用脫節
- 哪一關失敗後我知道自己錯在哪
- 哪一關失敗後我不知道該怎麼改進
- 哪一段關卡節奏讓我感到自然進步
- 哪一段關卡節奏讓我感到壓力突然暴增

例如：

> 第 3 關首次要求同時顧及左右兩線時，理解負擔上升過快。  
> 並不是敵方太強，而是玩家尚未建立穩定的多單位協同觀念。

又例如：

> 第 5 關雖然難度提升明顯，但因前一關已先鋪墊假動作概念，因此失敗後仍有明確學習感。

這種回饋比單純的「第 5 關通關率 32%」更接近設計問題本身。

---

## 關卡設計 Agent 如何使用這些心得

關卡設計 Agent 不需要直接讀取原始遊戲畫面來理解一切。  
它可以讀取：

- 遊玩記錄
- 攻略
- 心得
- 失敗樣式摘要
- 多次遊玩後的共通抱怨點與卡點

再進一步做微調建議。

這些建議可以是：

- 將某個新機制提前一關獨立介紹
- 把原本同一關首次出現的兩個概念拆開
- 在高壓關前加一個緩衝關
- 降低某個敵方配置的首次出現密度
- 增加某個關鍵提示的可讀性
- 讓某類失敗更容易被玩家理解原因

重點是：

關卡設計 Agent 不一定要直接「設計整個新遊戲」，  
而是可以先從 **微調既有關卡流程** 開始。

這樣風險小，也比較符合目前 agent 能力的實際上限。

---

## 一個可行的閉環

在 MUST 遊戲中，可以建立以下閉環：

1. LLM 玩家遊玩關卡
2. 產出攻略
3. 產出心得
4. 分析多次心得中的共通模式
5. 關卡設計 Agent 根據模式提出微調建議
6. 人類設計師審核
7. 更新關卡後再重跑測試

這是一個比「完全自動設計關卡」更現實、也更穩健的方向。

---

## 現在的 agent 夠用了嗎

答案是：

**夠做原型，但還不夠成熟到能完全自動接管心流設計。**

### 已經夠用的部分

目前的 agent 已經足以拿來做：

- 遊玩後的語言化回饋
- 攻略整理
- 心得摘要
- 多場測試後的問題歸納
- 小幅關卡微調建議

也就是說，拿來做概念驗證與半自動設計輔助，是可行的。

### 還不夠穩的部分

但如果要直接交給 agent 負責：

- 完整關卡節奏曲線設計
- 長篇章難度遞增安排
- 穩定保證心流成立
- 精準處理不同玩家程度下的主觀感受差異

那現在還不夠可靠。

原因是心流本身不是單一指標，而是一個混合現象，涉及：

- 規則理解速度
- 壓力承受度
- 失敗可解釋性
- 學習感
- 節奏起伏
- 主觀情緒反應

LLM 很適合描述這些東西，但未必已經足夠穩定地評估它們。

---

## 現階段最合理的定位

因此，現階段比較好的定位不是：

> 讓 agent 全自動做關卡設計

而是：

> 讓 agent 成為能玩、能寫攻略、能寫心得、能提出微調建議的高階設計輔助者

也就是把 agent 放在：

- 玩家測試者
- 體驗評論者
- 策略總結者
- 關卡微調提案者

的位置，而不是最終拍板者。

人類設計師仍然應該保有最後的節奏判斷權。

---

## MUST 遊戲中的心流設計重點

若要讓 agent 協助 MUST 關卡設計，應特別關注以下幾種體驗訊號：

- 新機制一次出現幾個才不會太亂
- 玩家在第幾關開始需要真正的多單位協同
- 哪些關卡是在教理解，哪些關卡是在測應用
- 哪些失敗能帶來學習感，哪些只會帶來挫折感
- 哪些高壓關前需要安排緩衝與整理
- 哪些資訊應透過 UI、提示或解說補強

這些訊號若能被穩定收集、整理與利用，MUST 遊戲的關卡曲線就有機會逐步逼近理想心流。

---

## 簡短總結

在 MUST 遊戲中，LLM 不只可以留下攻略，也應該留下心得。

攻略幫助系統理解：

> 怎麼玩比較好

心得幫助系統理解：

> 玩起來感覺怎麼樣

而關卡設計 Agent 讀取這些心得後，就可以針對關卡流程、機制引入節奏與難度曲線做出微調建議。

目前的 agent 已經足夠支撐這種 **半自動的心流調整原型**，  
但還不足以完全自動接管整體關卡難度設計。

因此，最實際的路線是：

> 讓 LLM 成為 MUST 遊戲中的玩家、評論者與微調提案者，  
> 再由人類設計師進行最後審核與取捨。
