# LLM 的經驗沉澱，為何可以被稱為「攻略」

## 問題意識

這個專案背後的一個核心問題是：

當一個模型玩過大量遊戲、經歷過大量訓練之後，它到底留下了什麼？

對於強化學習（RL）來說，這個問題相對明確。模型留下來的通常是數值化的策略痕跡，例如：

- Q-table 中各狀態與動作的價值
- DQN 或 policy network 的權重
- 對不同行動在不同局面下的偏好分布

也就是說，RL 留下的是一組壓縮在參數或數表中的經驗。這些經驗對決策有用，但通常不直接對人類可讀。

## LLM 留下的是什麼

如果把 LLM 用在遊戲決策上，情況就不太一樣。

LLM 不只是從抽象的 `action0`、`action1`、`action2` 和 reward 之間的對應關係中慢慢試錯。它還可以在進入遊戲前，就透過文字描述理解規則、角色、風險、非對稱情報與可能的欺騙結構。

例如：

- 如果遊戲描述中說「有殺手潛伏在陰影中」，LLM 即使在可觀察狀態欄裡還沒看到任何敵人，也可能先推理出貿然前進的風險
- 如果規則描述中指出某角色常透過假動作誤導對手，LLM 可以在真正看到行動前就先保留懷疑
- 如果某個玩家在歷史對局中經常口頭合作、結算時卻背叛，LLM 可以把這種行為模式提煉成後續對局的策略依據

這表示 LLM 的經驗不一定只存在權重裡，也可以被外顯為語言化的策略知識。

## 最精準的詞：攻略

當這些遊玩經驗被整理、提煉、重組之後，我們其實早就有一個非常精準的詞來描述它：

**攻略**

這個詞很重要，因為它比「記憶」、「筆記」或「摘要」更準確。

攻略不是單純記錄過去發生了什麼，而是把過去的經驗轉化成面向未來決策的知識。它會包含：

- 對規則的理解
- 對局勢的判讀
- 對對手或隊友的行為模型
- 對風險的提醒
- 對協作與承諾是否可信的判斷
- 對特定局面的行動建議

例如，下面這種內容就已經不是原始資料，而是攻略：

> 與某帳號組隊時，不可只相信口頭承諾；若對方聲稱要同步施法，仍需預留其臨時變招的風險。

## 與 RL 經驗的本質差異

RL 與 LLM 都可以從經驗中形成策略，但兩者沉澱經驗的形式不同。

- RL 把經驗壓縮成數值參數
- LLM 可以把經驗整理成語言化攻略

這裡最大的差別不只是表達形式不同，而是：

**LLM 的經驗在理想情況下，是人類也看得懂的。**

RL 的 Q 值、權重、value function 對機器有用，但對人類通常不透明。人類很難直接從一組參數中讀出：

- 為什麼這個行動比較好
- 它在怕什麼風險
- 它對某個玩家形成了什麼印象
- 它是如何理解這個局面的

但如果 LLM 把經驗寫成攻略，那麼人類就有機會直接閱讀這些內容，理解模型到底學到了什麼。

## 可讀性，是 LLM 經驗的一個關鍵價值

到了這個層次，LLM 經驗與 RL 數據最大的差別之一，就是可讀性。

RL 的經驗通常是：

- 可用
- 可優化
- 但不可直接閱讀

LLM 的經驗理想上則可以是：

- 可用
- 可閱讀
- 可批改
- 可教學
- 可傳遞
- 可與人類共同編修

但這裡有一個重要前提：

**前提是 LLM 真的有把經驗寫成人類看得懂的攻略。**

如果它為了壓縮資訊，最後發展出只有自己看得懂的黑話、縮寫或私有符號系統，那麼它雖然可能保留了資訊，卻失去了「可與人共享」的價值。

因此，LLM 經驗沉澱的研究，不只是看它有沒有留下知識，也要看它留下來的知識是否仍然對人類可理解。

## 攻略的另一個優勢：泛化

除了可讀性之外，攻略還有另一個非常重要的特性：

**它天然具有一定程度的泛化能力。**

這是因為攻略是以文字和語意的形式存在，而不是被綁死在某一個固定的狀態編號、動作編號或參數空間上。

RL 的經驗往往強依附於特定問題設定：

- 特定地圖
- 特定狀態表示
- 特定動作空間
- 特定 reward 結構

一旦換關卡、換規則、換觀測方式，原本的數值經驗通常不容易直接搬用。

但 LLM 的攻略如果寫得夠好，往往可以保留在更高層次的語意上。例如它記住的不是：

> 在某張地圖的某個 state index 裡，選 action 4 的期望值比較高。

而是：

> 當敵人具有假動作或延遲爆發能力時，不要只根據表面前搖交出全部防禦資源。

這樣的知識就不只適用於單一地圖，也可能適用於：

- 新的一關
- 相似但不完全相同的敵人
- 另一個有類似博弈結構的遊戲

換句話說，攻略可以透過語意類比進行遷移。

## 跨關卡、跨對手、跨遊戲的遷移

如果一個 LLM 已經累積出一批攻略，那麼在面對新局面時，系統可以直接把這些舊經驗以語言方式重新引入。

例如：

- 這一關與你之前玩過的第 7 關相似，但敵方移動速度更快
- 這名玩家的承諾風格接近你之前遇過的某帳號，需提防口頭合作與實際行動不一致
- 這個任務的資訊不完全結構，和《法師鬥惡龍》中的同步承諾博弈相似

這些提示的本質，就是把過往攻略當成可檢索、可引用、可比對的知識資產。

因此，LLM 的經驗不只是「保存下來」，而是可以在新環境中被重新解釋與重新套用。

這也使得 LLM 的學習方式更接近人類玩家。

人類玩家通常不會記住一整套精確的數值表，而是會記住：

- 這類敵人常用假動作
- 這種地形容易被包夾
- 這種隊友不可靠
- 這類資源不能太早交掉
- 這個新關卡很像以前某一關，只是節奏更快

這些本質上都是可以遷移的攻略。

## 從可讀，到可遷移

因此，攻略的價值不只在於人類能看懂，還在於它能跨情境複用。

可以把這個特性濃縮成一句話：

> LLM 的經驗不只可讀，還可類比；不只可保存，還可泛化。

這也是語言化經驗相較於純數值經驗的一大優勢。  
它不只是一種記錄過去的方法，也是一種幫助未來快速理解新局面的工具。

## 訓練成果如何體現

如果接受「LLM 留下的是攻略」這個觀點，那麼模型有沒有經過訓練、訓練品質好不好，就可以從攻略文本的質與量來觀察。

沒經過足夠訓練的 LLM，可能會出現：

- 攻略很少
- 攻略空泛
- 攻略只是在重述規則
- 攻略缺乏可操作性
- 攻略無法泛化到相似局面

訓練得比較好的 LLM，則可能表現為：

- 能穩定累積大量有效攻略
- 能從多局對戰中提煉出可重用的策略規則
- 能建立對不同玩家或敵人的行為模型
- 能辨識承諾、欺騙、合作失敗與資訊不對稱
- 能把這些經驗轉寫成之後可直接復用的語言知識

換句話說，訓練成果不只體現在模型參數上，也體現在它是否能產出高品質的攻略。

## 一個可操作的研究命題

因此，這個方向可以被濃縮成下面這個研究命題：

> RL 將經驗壓縮成參數；LLM 可以將經驗整理成攻略。

而進一步的問題則是：

1. LLM 能不能從遊戲中累積有效經驗？
2. LLM 能不能把這些經驗提煉成可復用的攻略？
3. 這些攻略是否對後續決策真的有幫助？
4. 這些攻略是否仍然對人類可讀、可檢查、可修正？

如果答案是肯定的，那麼 LLM 的遊戲訓練成果就不再只是黑箱中的權重變化，而是一種能夠被閱讀、比較、編輯與傳承的策略資產。
