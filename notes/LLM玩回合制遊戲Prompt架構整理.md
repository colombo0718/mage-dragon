# LLM 玩回合制遊戲 Prompt 架構整理

## 一、LLM 玩《法師鬥惡龍》的 Prompt 架構

目前適合的 prompt 結構為：

1. 遊戲規則 (Rules)  
2. LLM 角色 / 個性 (Persona)  
3. 最近對話 (Recent Dialogue)  
4. 當前狀態 (Current State)  
5. 回應規則 (Output Rule)

---

## 1. 遊戲規則

這一段是固定模板，每回合都需要帶入。

重點不是完整教學，而是讓 LLM 知道：

- 遊戲目標
- 可用動作
- 動作效果
- 勝負條件
- 特殊限制

例如：

- 你是法師B，隊友是法師A  
- 法師HP上限 5，魔龍HP上限 20  

可用動作：

- 0 = 無  
- 1 = 閃電  
- 2 = 護盾  
- 3 = 補血  

技能機制：

- 閃電 / 護盾 / 補血需要詠唱1回合，下一回合施放  
- 兩人同時護盾 → 完全抵擋傷害  
- 單人護盾 → 傷害減半並由兩人平分  
- 補血只能補隊友  

魔龍可能狀態：

- 無  
- 吸氣  
- 噴火  
- 抬手  
- 爪擊  

吸氣後可能噴火，也可能停。  
抬手後可能爪擊，也可能停。

勝負條件：

- 戰敗：兩名法師死亡，兩人 -10 分  
- 戰勝：個人得分 = 自己造成的傷害

---

## 2. LLM 角色 / 個性

這一段不是必要，但可以讓 LLM 的決策風格更穩定。

例如：

- 偏好輸出的玩家：只要不是明顯會輸，優先閃電攻擊  
- 保守玩家：優先保命與護盾  
- 心機玩家：可能會誤導隊友以爭取輸出  

這樣可以讓 AI 有「性格」。

---

## 3. 最近對話

只需要保留 **本回合相關的對話**。

不需要整局歷史。

建議保留：

- 最近 1~4 句
- 本回合策略協調
- 尚未完成的承諾

例如：

你：龍在吸氣，我建議一起開盾  
隊友：我這回合幫你補血  
你：那我準備輸出

---

## 4. 當前狀態

這是 LLM 做決策的核心資訊。

至少包含：

- 你的HP
- 隊友HP
- 魔龍HP
- 你的狀態
- 隊友狀態
- 魔龍狀態

例如：

你的HP：4  
隊友HP：3  
魔龍HP：12  

你的狀態：無  
隊友狀態：無  
魔龍狀態：吸氣

---

## 5. 回應規則

建議最簡單的方式：

- **回數字 = 出招**
- **回文字 = 討論**

例如：

如果你已經決定出招，請只回覆一個數字：

0  
1  
2  
3  

如果你想先和隊友討論策略，請回覆一段想跟隊友說的文字。

限制：

- 不要解釋規則
- 不要輸出其他格式
- 不要講思考過程

# 二、LLM 與人類玩回合制遊戲的通用 Prompt 架構

這個架構可以套用到任何回合制遊戲。

## 通用結構

1. 遊戲規則 (Rules)  
2. LLM 身份 / 個性 (Persona)  
3. 最近對話 (Dialogue)  
4. 當前局面 (Game State)  
5. 輸出規則 (Output Format)

---

## 1. 遊戲規則

描述：

- 遊戲目標
- 可用動作
- 動作效果
- 勝負條件

規則應該是 **短版規則卡**，而不是完整教學文件。

---

## 2. LLM 身份 / 個性

說明 AI 的：

- 角色立場（隊友 / 對手）
- 行為傾向（保守 / 進攻 / 欺騙 / 合作）
- 策略目標（求勝 / 求分 / 求生）

這能讓 AI 行為更穩定且更像玩家。

---

## 3. 最近對話

如果遊戲允許協商或欺騙，就需要這一段。

原則：

- 只保留最近討論
- 不帶整局歷史
- 保留承諾或試探

---

## 4. 當前局面

所有與決策相關的 state。

可能包含：

- 血量 / 能量
- 手牌
- 棋盤位置
- 單位狀態
- 回合數
- 可觀察環境

目標是讓 LLM **只靠當前 prompt 就能決策**。

---

## 5. 輸出規則

必須明確限制輸出。

常見設計：

### 模式一：純行動
只輸出動作編號

### 模式二：行動 + 溝通
- 數字 = 行動
- 文字 = 討論

---

# 三、核心原則

LLM 玩回合制遊戲時，每次推理應提供：

- 規則 (Rules)
- 角色 (Persona)
- 當前局面 (State)
- 當局對話 (Dialogue)
- 輸出規則 (Output Rule)

不建議帶入：

- 全局歷史狀態
- 全部對話紀錄
- 過長背景敘事

因為這會：

- 增加 token 成本
- 稀釋注意力
- 對當前決策幫助不大
